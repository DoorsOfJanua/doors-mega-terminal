TIPS & TRICKS
Route models by task. Haiku for grunt work (installing, running tests, file ops). Sonnet for feature building and code review. Opus for architecture, hard debugging, and creative work. Save ~50% in credits.
CLAUDE.md is your autopilot. Put project conventions, build commands, and coding style in CLAUDE.md at your project root. Every Claude session reads it automatically. The better your CLAUDE.md, the less you repeat yourself.
Use /compact before you hit the wall. When context gets long, Claude starts forgetting. Run /compact proactively to compress history and keep the session sharp. Better to compact early than to lose important context.
Pipe for automation. Chain Claude into scripts: git diff | claude -p "review this" or cat error.log | claude -p "what went wrong?". Great for CI pipelines and automated reviews.
The snake means done. When a terminal window shows the glowing border animation, that window's task has finished. Start typing to dismiss it. Sounds play only on task completion.
Workspaces keep you sane. Group related projects into workspaces using the tabs. One for "frontend", one for "backend", one for "side projects". Double-click to rename, right-click to delete. Everything saves between sessions.
SSH into remote and run Claude there. Claude Code works over SSH. Open a terminal, SSH into your server, run claude. You now have a remote AI dev session with full file access on that machine.
Run Claude headless in the background. Use claude -p "do X" --output-format json for fire-and-forget tasks. Combine with nohup or tmux for long-running background agents.
Multiple Claudes, one repo. Use Claude Squad (cs) or git worktrees to run parallel sessions on the same project without file conflicts. Each session gets its own branch and working directory.
Sessions survive restarts. DMT saves your open windows, positions, sizes, and workspaces. Close and reopen, everything comes back exactly as it was.

GUIDE

WORKSPACES

Click tab to switch workspace
Click + to create a new workspace
Double-click tab to rename it
Right-click tab to delete it
Each workspace has its own set of terminal windows.

TERMINAL WINDOWS

+ ADD creates a new terminal window
Drag header to move window
Drag edges to resize
TILE auto-arranges all windows in a grid
− + buttons on window header change that window's font size

CLAUDE BUTTON

Each window has a Claude button that types claude + Enter into the terminal to launch Claude Code.

MODEL BADGE

Click the Sonnet/Haiku/Opus badge on a window header to switch models. This sends /model <name> directly to the terminal.

SIDE PANELS

Left-click a side panel to cycle through animations (panel, warp, cockpit, nebula, matrix, radar, fractal).
Right-click to cycle through background images (Star Wars cockpit panels). Not available in Classic mode.

COMMAND CENTER

` (backtick) toggles the command center open/closed.
− + next to SETTINGS changes command center text size.
SETTINGS menu has appearance, colors, sound, shortcuts, and this guide.

KEYBOARD

1-9 focus window by number
` (backtick) toggle command center
Ctrl+Shift+C open Claude Code in focused window
